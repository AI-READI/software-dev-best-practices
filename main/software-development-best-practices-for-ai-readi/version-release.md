# 6. Version release

* 6.1. Document changes between versions of your software in a "CHANGELOG.md" file located at the root of your GitHub repository [Category: C] [Ref: [1](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-1)]
   * 6.1.1. Follow the Semantic Versioning 2.0 principles outlined [here](https://semver.org) for version numbering [Category: C] [Ref: [1](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-1)]
   * 6.1.2. Follow the "[keep a changelog](https://keepachangelog.com/en/1.0.0/)" guidelines for structuring your CHANGELOG file [Category: C] [Ref: [1](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-1)]
* 6.2. Make a GitHub release for each version of your software [Category: A] [Ref: [1](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-1)]
   * 6.2.1. Use an automated git changelog builder based on conventional commits to document the changes for each release [Category: D] [Ref: [1](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-1)]
* 6.3. Deposit in a language-specific repository if available: [Category: A] [Ref: [2](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-2)]

Docker-based tools: [Dockstore](https://dockstore.org/) [Category: A] [Ref: [2](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-2), [5](https://github.com/AI-READI/software-development-best-practices/blob/code-development/main/software-development-best-practices-for-ai-readi/overview-and-resources.md#user-content-5)]
